gridify
code tests docs

Minimal and lightweight table generator. It has four main focuses to its design philosophy.

  • Having an easy and intuitive syntax. The principle of least surprise is in full effect. When possible, grid configuration properties are named after their native html/js counterparts.
  • Not attempting to do everything. Gridify will avoid adding a .width property to column definitions since that is something that can easily be done through CSS. It relies on the developer's knowledge of the base web technologies rather than knowledge of gridify itself.
  • Generating readable markup. The generated markup is intended to resemble something a developer would type-out. It can be easily read and programmatically manipulated by a developer.
  • Giving the developer full control. Event hooks execute at every stage of table creation, giving the developer full control over how the table is built as it is being built.
